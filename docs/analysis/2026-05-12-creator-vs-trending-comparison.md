# 떡상 탐지 vs 크리에이터 인사이트 비교

> 코드 grep 기반 사실. 추측 없음.

---

## 1. 비교 표

| 항목 | 🔥 떡상 탐지 | 💡 크리에이터 인사이트 |
|------|-------------|----------------------|
| **URL** | `/video-rankings` (pageMode=viral 토글) | `/creator-insights` (독립 페이지) |
| **진입 방법** | 비디오 랭킹 페이지 내 탭 전환 | 사이드바 직접 접근 |
| **데이터 소스** | `viralboard_data` (실시간 수집본) | `viral_title_archive` (90일 아카이브) |
| **정렬 기준** | `views / subscriber_count` — 클라이언트 계산 | `viral_ratio` DB 컬럼 — 아카이브 시점 계산 |
| **최소 임계값** | `subscriber_count > 0` (임계값 없음) | `viral_ratio >= 5` |
| **국가 필터** | KR/US/BR/JP 선택 가능 | KR 고정 |
| **기간 필터** | Daily(1d) / Weekly(7d) / Monthly(30d) | 7일 / 30일 / 90일 |
| **채널 크기** | 없음 | 마이크로(1K~10K) / 일반(10K~1M) |
| **영상 타입** | Shorts/Long 구분 없음 | 가로 / Shorts 선택 |
| **카드 형식** | 테이블 (rank, views, viralScore 배지) | 리스트 + 상세 모달 |
| **댓글 분석** | ❌ | ✅ (YouTube Comments API) |
| **신선도 표시** | ❌ | 🔥 HOT NOW / 📈 TRENDING / ✅ PROVEN |
| **결과 limit** | TOP 50 (`viralScore` 순) | TOP 100 (`viral_ratio` 순) |

---

## 2. 핵심 차이 (한 줄 요약)

- **떡상 탐지**: "지금 이 순간 뭐가 터지고 있나?" — 실시간 데이터, 국가별 비교 가능
- **크리에이터 인사이트**: "어떤 콘텐츠 공식이 역사적으로 통했나?" — 아카이브 데이터, 채널 규모별 전략 참고용

두 페이지는 **같은 viral 메트릭을 공유하지만 시간축이 다름**:
- 떡상 탐지 = 현재 스냅샷 (`viralboard_data`)
- 크리에이터 인사이트 = 과거 성공 패턴 (`viral_title_archive`)

---

## 3. 현재 혼란 원인

1. 사이드바 레이블이 기능을 설명하지 않음 (`nav_creator_insights` 번역키만)
2. 떡상 탐지는 `/video-rankings` 내부에 숨어 있어 발견성 낮음
3. 두 페이지 모두 "바이럴" 영상 보여주는 것처럼 보여서 중복 오해

## 4. 옵션 분석

### 옵션 A — 라벨/설명 추가 (15분)

**변경 내용:**
- 크리에이터 인사이트 페이지 상단: 한 줄 설명 추가
  > "📚 과거 90일 바이럴 성공 패턴 아카이브 — 콘텐츠 기획 참고용"
- 떡상 탐지 탭 상단: 한 줄 설명 추가
  > "⚡ 실시간 탐지 — 지금 구독자 대비 조회수가 폭발 중인 영상"
- 사이드바 `nav_creator_insights` 번역키 설명 보강

**장점**: 코드 변경 최소, 즉시 적용
**단점**: 근본적인 탐색 구조는 그대로 (발견성 문제 미해결)

---

### 옵션 B — 구조 통합 (60분)

**방향 A**: 떡상 탐지 → 크리에이터 인사이트로 흡수
- 크리에이터 인사이트 상단에 "실시간" / "아카이브" 탭 추가
- `/creator-insights?mode=realtime` 형태로 분기

**방향 B**: 크리에이터 인사이트 → 독립 유지, 떡상 탐지 사이드바 직접 노출
- 현재 `/video-rankings` 내부 토글 → 별도 route `/viral-detect` 분리

**장점**: UX 명확화, 중복 제거
**단점**: 라우터/사이드바 수정 필요, 회귀 위험

---

## 5. 권장안

**옵션 A 채택 권장.**

이유:
1. 두 페이지의 **사용자 의도가 근본적으로 다름** — 통합 시 오히려 혼란
2. 떡상 탐지는 국가 비교 + 실시간, 크리에이터 인사이트는 KR 심층 분석 — 병존이 맞음
3. 옵션 B는 라우터·사이드바 영향 범위가 커서 별도 사이클 필요
4. 라벨만 추가해도 P5 분노("탭 의미 불명") 해소 가능
