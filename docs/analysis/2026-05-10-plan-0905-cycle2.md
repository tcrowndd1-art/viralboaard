### G1.5.1 Niche/CPM 진단
[[2026-05-10-plan-0905-1차-v4#G2]] 잔여:
- DLL `0xC000013A` (STATUS_CONTROL_C_EXIT) — venv 정상, import 정상
- 가설: ViralBoard_Niche (03:00) ↔ Fetcher (30분 주기) 시간 충돌 / main() 외부 kill
- **검증 방법**: 03:00 트리거 시점 로그 + 동시 작업 충돌 추적

```powershell
# 다른 ViralBoard_* 작업의 03:00 시점 활동 확인
Get-WinEvent -LogName Microsoft-Windows-TaskScheduler/Operational -MaxEvents 200 |
  Where-Object { $_.TimeCreated.Hour -eq 3 -and $_.Properties[0].Value -like "*ViralBoard*" }
```

### G1.5.2 Fetcher cron 30분 → 6시간 검토
5/8 quota-diag 보고서 6장 권고:
- ViralBoard_Fetcher 30분 → **6시간** (48회 → 4회). 절감: -123K u/day (-92%)
- ViralBoard_Fresh 1시간 → **3시간** (24회 → 8회). 절감: -38K u/day (-67%)
- 합계 절감: ~162K/day → 신규 일일 사용량 ~40K (한도 70K 안전)

**전제**: cycle1 G2.5 (invalid_combos cache) + G2.6 (SECONDARY 6h gate) 효과로 **현재 사용량 ~14.4K/day**. 따라서 6시간/3시간 cron 조정해도 안전.

### G1.5.4 (Gemini patch v1.1) — Windows TS 실패 알림 추가
**누락 보강**: 마스터 plan Ch 6 cron 5트랙 중 GitHub Actions만 텔레그램 알림(cycle1 G1). Windows TS는 알림 0.

**해결책**:
- run_fetcher.ps1 / run_fresh.ps1 끝에 텔레그램 webhook step 추가
- exit code !=0 시 curl POST telegram (TELEGRAM_BOT_TOKEN/CHAT_ID = .env 사용)
- 또는 별도 wrapper script (run_with_alert.ps1)

```powershell
# PowerShell 스니펫 (각 run_*.ps1 끝에 추가)
if ($exitCode -ne 0) {
    $msg = "[VBoard TS FAIL] $($MyInvocation.MyCommand.Name) exit=$exitCode"
    $body = @{ chat_id = $env:TELEGRAM_CHAT_ID; text = $msg } | ConvertTo-Json
    Invoke-RestMethod -Uri "https://api.telegram.org/bot$env:TELEGRAM_BOT_TOKEN/sendMessage" -Method Post -Body $body -ContentType "application/json" -ErrorAction SilentlyContinue
}
```
